# Postgres Explorer

A fast PostgreSQL desktop explorer — visual browsing, filtering, joins, and
query building with always-visible generated SQL. Built to feel like TablePlus,
not pgAdmin. No Electron, no telemetry, no cloud.

**Stack:** Tauri 2 · React + TypeScript · TailwindCSS · TanStack Table/Virtual ·
Monaco · Rust · `postgres` (r2d2 pool, native-tls).

> Sibling project to `sqliteviewer` — same UI and architecture, retargeted from
> SQLite to PostgreSQL. The pure-Rust core crate is `pgcore` (vs `dbcore`).

## Architecture

```
React (UI only)  ──invoke()──▶  Tauri commands  ──▶  pgcore (pure Rust)
 Zustand stores                  (src-tauri)          schema · sql · pagination
```

- **`crates/pgcore`** — all database access and SQL generation, with **no UI/Tauri
  dependency**. Pure Rust + the `postgres` client, so its SQL-building logic is
  fully unit-testable without the system GUI libraries (`cargo test -p pgcore`).
- **`src-tauri`** — thin Tauri command layer over `pgcore`; owns connection state
  (an r2d2 pool) and the IPC surface.
- **`src`** — React frontend. Never touches PostgreSQL; it only sends typed
  requests and renders typed responses. `src/ipc/types.ts` mirrors the Rust DTOs.

Tables are always **paginated** (`LIMIT … OFFSET …`) — entire tables are never
loaded — and the visual *browser* only ever generates `SELECT`s. User values are
**always bound** as `$N` parameters, never interpolated.

Editing is opt-in and explicit (it's never auto-generated by the builder):

- **Edit a cell** — double-click a cell in the grid; commits a parameterized
  `UPDATE … WHERE <primary key>`. Enabled only for a single-table view whose
  table has a primary key.
- **Add / drop a column** — from the Columns panel; runs `ALTER TABLE`. Dropping
  asks for inline confirmation. The column *type* can't be parameterized, so it
  passes a strict allow-list before interpolation.

### How it differs from the SQLite version

- **Connecting** is a form (host / port / database / user / password / SSL mode)
  with locally-saved connection profiles, not a file picker.
- **Schema discovery** reads `information_schema` / `pg_catalog` instead of
  `sqlite_master` + PRAGMAs. Objects in `public` are shown bare (`users`); other
  schemas are qualified (`sales.orders`).
- **Types** are static: cell values map PostgreSQL types (bool, int2/4/8,
  float4/8, numeric, text, json/jsonb, uuid, timestamps, bytea, …) to a tagged
  union. NUMERIC is preserved as text; placeholders are `$1, $2, …`.
- **Row counts** use the planner's `reltuples` estimate for the instant figure,
  `COUNT(*)` for the exact one.
- Case-insensitive convenience filters (contains / starts-with / ends-with) use
  `ILIKE`.

## Prerequisites

- Node 20+ and `pnpm`
- Rust (stable) + Cargo
- **Linux system libraries** (Fedora):

  ```bash
  sudo dnf install -y webkit2gtk4.1-devel openssl-devel librsvg2-devel \
    libappindicator-gtk3-devel file curl wget gcc gcc-c++ make pkgconf-pkg-config
  ```

  (`openssl-devel` is required by `native-tls` for SSL connections. No
  PostgreSQL client libraries are needed — `postgres` is a pure-Rust driver.)

## Develop

```bash
pnpm install            # JS dependencies
pnpm tauri dev          # run the desktop app (Vite + Rust)
```

Other scripts:

```bash
pnpm typecheck          # tsc --noEmit
pnpm lint               # eslint
pnpm build              # production frontend build
cargo test -p pgcore    # SQL-builder unit tests (no GUI libs or server required)
```

### Live database tests

`crates/pgcore/tests/pg_integration.rs` exercises schema discovery and query
execution end-to-end. It is skipped unless a server is configured:

```bash
PGCORE_TEST_HOST=localhost PGCORE_TEST_USER=postgres \
PGCORE_TEST_DB=postgres cargo test -p pgcore
```

## Keyboard shortcuts

- `Cmd/Ctrl + O` — open the connection dialog
- `Cmd/Ctrl + B` — toggle the sidebar
- `Cmd/Ctrl + J` — toggle the SQL panel
